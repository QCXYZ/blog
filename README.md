### 接口文档

#### 1. 博客列表接口

- **接口描述：** 获取博客列表信息
- **接口URL：** `/admin/getBlogList`
- **请求方式：** POST
- **请求ContentType：** application/json
- **请求参数：**

| 参数名      | 必选 | 类型     | 说明   |
|----------|----|--------|------|
| pagenum  | 是  | int    | 页码   |
| pagesize | 是  | int    | 每页数量 |
| typeId   | 否  | Long   | 类型ID |
| title    | 否  | String | 博客标题 |

- **请求示例：**

```json
{
  "pagenum": 1,
  "pagesize": 10,
  "typeId": 2,
  "title": "示例标题"
}
```

- **成功返回值：**

| 参数名     | 类型      | 说明     |
|---------|---------|--------|
| success | boolean | 操作是否成功 |
| code    | int     | 状态码    |
| message | String  | 返回信息   |
| data    | Object  | 返回数据   |

- **返回示例：**

```json
{
  "success": true,
  "code": 200,
  "message": "获取博客列表成功",
  "data": {
    ...
  }
  // 返回数据的具体内容
}
```

#### 4. 删除博客接口

- **接口描述：** 根据博客ID删除博客
- **接口URL：** `/admin/blogs/{id}/delete`
- **请求方式：** GET
- **请求参数：**

| 参数名 | 必选 | 类型   | 说明   |
|-----|----|------|------|
| id  | 是  | Long | 博客ID |

- **请求示例：**

```
/admin/blogs/1/delete
```

- **成功返回值：**

| 参数名     | 类型      | 说明     |
|---------|---------|--------|
| success | boolean | 操作是否成功 |
| code    | int     | 状态码    |
| message | String  | 返回信息   |

- **返回示例：**

```json
{
  "success": true,
  "code": 200,
  "message": "博客删除成功"
}
```

### 注意事项

- 所有接口返回的`success`字段为`true`时表示操作成功，为`false`时表示操作失败。
- `code`字段表示操作的状态码，`200`表示成功，其他值表示具体的错误类型。
- 接口的`data`字段将包含具体的返回数据，其结构取决于具体的业务逻辑和需求。
- 请求参数中的`id`，`typeId`等字段应确保在调用前已经存在于数据库中，否则可能会导致操作失败。
- 对于`POST`请求，需要客户端设置`Content-Type`为`application/json`，并将请求体（body）中的数据序列化为JSON格式。

#### 1. 类型管理接口

### 概述

该部分接口用于管理博客文章的分类信息，包括新增、修改分类以及删除分类的功能。

#### 接口列表

1. **新增/修改分类**
2. **删除分类**

---

### 1. 新增/修改分类

- **接口描述：** 用于新增或修改文章的分类信息。
- **接口URL：** `/admin/types`
- **请求方式：** POST
- **支持格式：** JSON
- **请求参数说明：**

| 参数名  | 参数位置 | 类型   | 是否必须 | 说明           |
|------|------|------|------|--------------|
| type | body | Type | 是    | 分类对象，包含名称等信息 |

- **Type对象说明：**

| 字段名  | 类型     | 说明               |
|------|--------|------------------|
| id   | Long   | 分类ID，新增时为空，修改时必填 |
| name | String | 分类名称，新增和修改时都必须提供 |

- **成功响应：**

| 字段名     | 类型      | 说明       |
|---------|---------|----------|
| success | boolean | 操作是否成功   |
| code    | int     | 状态码      |
| message | String  | 操作结果描述   |
| data    | null    | 固定返回null |

- **错误响应：** 同成功响应，但`success`字段为`false`，并在`message`字段中提供错误信息。

- **请求示例（新增）：**

```json
{
  "type": {
    "name": "技术"
  }
}
```

- **请求示例（修改）：**

```json
{
  "type": {
    "id": 1,
    "name": "科技"
  }
}
```

- **成功返回示例：**

```json
{
  "success": true,
  "code": 200,
  "message": "修改成功",
  "data": null
}
```

---

### 2. 删除分类

- **接口描述：** 根据分类ID删除指定的文章分类。
- **接口URL：** `/admin/types/{id}/delete`
- **请求方式：** GET
- **请求参数说明：**

| 参数名 | 参数位置 | 类型   | 是否必须 | 说明    |
|-----|------|------|------|-------|
| id  | path | Long | 是    | 分类的ID |

- **成功响应：** 同上述新增/修改分类接口的成功响应。

- **错误响应：** 同上述新增/修改分类接口的错误响应。

- **请求示例：**

```
GET /admin/types/1/delete
```

- **成功返回示例：**

```json
{
  "success": true,
  "code": 200,
  "message": "删除成功",
  "data": null
}
```

#### 注意事项

- 所有接口返回的`success`字段为`true`时表示操作成功，为`false`时表示操作失败。
- `code`字段表示操作的状态码，`200`表示成功，其他值表示具体的错误类型。
- 新增分类时，系统将校验分类名称是否已存在，以防止重复添加。
- 修改分类时，除了校验名称的唯一性外，还需要提供分类的ID。
- 删除分类操作将直接根据ID执行删除，需确保操作前已经确认分类下没有博客文章或已经做好文章迁移的准备。
